# Internationalization (I18n) — Translations, Pluralization, Locale Switching

## Overview

Rails I18n (shorthand for internationalization) provides a framework to translate your application into different languages.

## Configuration

```ruby
# config/application.rb
config.i18n.default_locale = :en
config.i18n.available_locales = [:en, :fr, :es, :de, :ja]
config.i18n.fallbacks = true
config.i18n.load_path += Dir[Rails.root.join("config", "locales", "**", "*.{rb,yml}")]
```

## Translation Files

```yaml
# config/locales/en.yml
en:
  hello: "Hello world"
  greetings:
    morning: "Good morning"
    evening: "Good evening"
  articles:
    title: "Title"
    created: "Article was successfully created."
    updated: "Article was successfully updated."
    destroyed: "Article was successfully destroyed."
  helpers:
    label:
      article:
        title: "Title"
        body: "Body"
    submit:
      create: "Create %{model}"
      update: "Update %{model}"
```

```yaml
# config/locales/fr.yml
fr:
  hello: "Bonjour le monde"
  greetings:
    morning: "Bonjour"
    evening: "Bonsoir"
  articles:
    title: "Titre"
    created: "L'article a été créé avec succès."
```

## Using Translations

### In Views

```erb
<%= t("hello") %>
<%= t("greetings.morning") %>
<%= t("articles.title") %>
```

### In Controllers

```ruby
redirect_to @article, notice: t("articles.created")
flash[:alert] = t("articles.not_found")
```

### In Models

```ruby
class Article < ApplicationRecord
  validates :title, presence: { message: :required }
  # Looks up: en.errors.messages.required
end
```

### With Interpolation

```yaml
en:
  welcome: "Welcome, %{name}!"
  article_count: "You have %{count} articles."
```

```erb
<%= t("welcome", name: current_user.name) %>
<%= t("article_count", count: @articles.size) %>
```

## Setting Locale

### From URL Parameters

```ruby
# config/routes.rb
scope "(:locale)", locale: /en|fr|es|de|ja/ do
  resources :articles
  root "articles#index"
end
```

```ruby
# app/controllers/application_controller.rb
class ApplicationController < ActionController::Base
  before_action :set_locale

  private

  def set_locale
    I18n.locale = params[:locale] || I18n.default_locale
  end

  def default_url_options
    { locale: I18n.locale }
  end
end
```

### From Browser Preferences

```ruby
def set_locale
  I18n.locale = extract_locale_from_accept_language_header || I18n.default_locale
end

def extract_locale_from_accept_language_header
  request.env["HTTP_ACCEPT_LANGUAGE"]&.scan(/^[a-z]{2}/)&.first
end
```

### From User Settings

```ruby
def set_locale
  I18n.locale = current_user&.locale || I18n.default_locale
end
```

## Pluralization

```yaml
en:
  inbox:
    zero: "You have no messages"
    one: "You have 1 message"
    other: "You have %{count} messages"
```

```ruby
t("inbox", count: 0)   # "You have no messages"
t("inbox", count: 1)   # "You have 1 message"
t("inbox", count: 5)   # "You have 5 messages"
```

### Russian Pluralization

```yaml
ru:
  inbox:
    zero: "У вас нет сообщений"
    one: "У вас %{count} сообщение"
    few: "У вас %{count} сообщения"
    many: "У вас %{count} сообщений"
    other: "У вас %{count} сообщений"
```

## Localized Views

Rails supports locale-specific templates:
- `app/views/articles/index.en.html.erb`
- `app/views/articles/index.fr.html.erb`

Rails picks the template based on `I18n.locale`.

## Date and Time Localization

```yaml
en:
  date:
    formats:
      default: "%Y-%m-%d"
      short: "%b %d"
      long: "%B %d, %Y"
  time:
    formats:
      default: "%a, %d %b %Y %H:%M:%S %z"
      short: "%d %b %H:%M"
      long: "%B %d, %Y %H:%M"
```

```erb
<%= l(Date.today, format: :long) %>
<%= l(Time.current, format: :short) %>
<%= I18n.l(@article.created_at) %>
```

## Number Localization

```ruby
I18n.with_locale(:fr) do
  number_to_currency(1234.56)  # "1 234,56 €"
  number_to_percentage(75.5)   # "75,50%"
  number_with_delimiter(1234567)  # "1 234 567"
end
```

## ActiveRecord Error Messages

```yaml
en:
  errors:
    format: "%{attribute} %{message}"
    messages:
      blank: "can't be blank"
      taken: "has already been taken"
      too_short: "is too short (minimum is %{count} characters)"
      too_long: "is too long (maximum is %{count} characters)"
      wrong_length: "is the wrong length (should be %{count} characters)"
      not_a_number: "is not a number"
      greater_than: "must be greater than %{count}"
      inclusion: "is not included in the list"
      exclusion: "is reserved"
```

### Model-Specific Messages

```yaml
en:
  activerecord:
    models:
      article: "Article"
      comment: "Comment"
    attributes:
      article:
        title: "Title"
        body: "Body"
      comment:
        body: "Comment"
    errors:
      models:
        article:
          attributes:
            title:
              blank: "Please enter a title"
```

## I18n.with_locale

```ruby
I18n.with_locale(:fr) do
  # all translations within this block use French
  t("hello")  # "Bonjour le monde"
end
```

## Fallbacks

```ruby
# config/application.rb
config.i18n.fallbacks = [:en]

# If a translation is missing in the current locale, fall back to :en
```

## Testing I18n

```ruby
require "test_helper"

class I18nTest < ActiveSupport::TestCase
  test "available locales" do
    assert_includes I18n.available_locales, :en
    assert_includes I18n.available_locales, :fr
  end

  test "translation exists" do
    I18n.with_locale(:en) do
      assert_equal "Hello world", t("hello")
    end
    I18n.with_locale(:fr) do
      assert_equal "Bonjour le monde", t("hello")
    end
  end
end
```

## Organization

For large applications, split translation files:

```
config/locales/
├── en.yml              # top-level keys
├── en/
│   ├── articles.yml    # article-specific
│   ├── users.yml       # user-specific
│   ├── models.yml      # model names and attributes
│   └── errors.yml      # error messages
├── fr.yml
└── fr/
    ├── articles.yml
    ├── users.yml
    ├── models.yml
    └── errors.yml
```

```ruby
# config/application.rb
config.i18n.load_path += Dir[Rails.root.join("config", "locales", "**", "*.{rb,yml}")]
```
