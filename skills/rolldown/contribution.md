## Contribution Guide

### Open Development

Rolldown is developed in the open on [GitHub](https://github.com/rolldown/rolldown). All work happens in public repositories, and the team communicates through GitHub issues, discussions, and pull requests.

### AI Usage Policy

- AI-generated pull requests are welcome if they are of high quality and the contributor takes responsibility for the code
- AI should be used as a tool to assist, not replace, human judgment
- All AI-generated code must be reviewed and understood by the submitter
- Test cases must be included with AI-generated changes

### Reporting a Bug

1. Search existing issues to avoid duplicates
2. Use the bug report template
3. Include: Rolldown version, Node.js version, OS, minimal reproduction
4. Provide expected vs actual behavior

### Requesting New Functionality

1. Open a GitHub Discussion first to gauge interest
2. Explain the use case and expected behavior
3. Consider whether it fits Rolldown's scope

### Submitting a Pull Request

**Send a pull request directly** for small fixes (typos, minor bugs, documentation improvements).

**Discuss the approach first** for larger changes by opening a discussion or issue.

**Draft pull requests** are welcome for work-in-progress to get early feedback.

**Branch organization**:
- Fork the repository
- Create a feature branch from `main`
- Keep PRs focused — one feature/fix per PR
- Ensure CI passes before requesting review

See: [Contribution Guide](https://rolldown.rs/contribution-guide/)
